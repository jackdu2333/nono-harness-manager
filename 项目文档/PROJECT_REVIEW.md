# Nono Harness Manager 项目复盘与架构文档

**创建时间**: 2026-06-25

## 1. 项目定位与核心需求 (Project Overview)
**Nono Harness Manager** 是一个基于桌面端的统一智能资产控制台。
它的核心任务是作为大语言模型客户端（Agents）、多上下文协议服务（MCP Servers）、记忆（Memory）以及技能（Skills）的统一管理后台。
- **管理台属性**：MCP 和 Agents 页面**不是简单的配置阅读器**，而是**高信息密度的资产管理台**。
- **直觉与效率**：用户第一眼就能看清：有哪些客户端、是否可用、配置在哪、出了什么问题。
- **类 macOS 原生体验**：拒绝网页后台模板感，追求原生桌面应用级别的高级交互感（如 Split View 分栏、毛玻璃、微动画、拖拽排序）。

## 2. 架构与技术选型 (Architecture & Tech Stack)
- **底层驱动 (Backend)**: Tauri (Rust), SQLite (sqlx 驱动)
- **前端框架 (Frontend)**: React 19 + Vite + TypeScript
- **状态管理**: Zustand
- **UI & 样式**: Tailwind CSS, Shadcn UI, Radix UI 组件库
- **交互强化**: `@dnd-kit` (实现高性能拖拽排序), `lucide-react` (图标)

## 3. 开发规范与核心原则 (Development Principles)
在后续的所有开发过程中，必须**严格遵守**以下原则：
1. **优先最小充分方案**：不添加未被明确要求的功能、过度抽象、灵活性或冗余的可配置性。
2. **非必要不改库 (No DB Schema Changes)**：坚决避免修改现有的 SQLite 数据库 Schema。如果遇到必须要增加的新状态（如拖拽排序、屏蔽删除），**优先利用现有字段（如 `status`）或使用前端本地存储（`localStorage`）绕过数据库修改**。
3. **精准修改与收敛**：每一行改动都应能追溯到用户请求，绝不顺手重构、格式化或删除无关内容。
4. **视觉与审美红线**：UI 必须克制且高级。消除所有不必要的横向滚动条、过重的纯黑分割线、巨大突兀的按钮。选中态应通过淡淡的背景色和侧边高亮线来体现，而非生硬的边框。

## 6. 踩坑点与经验教训 (Pitfalls & Gotchas)

> **坑点 1：Tauri 劫持浏览器原生弹窗**
> **表现**：前端调用 `window.confirm()` 进行删除二次确认时，页面没有任何反应，且中断了后续代码执行。
> **原因**：Tauri 出于桌面端安全机制，默认禁用了浏览器的 HTML dialog 接口。
> **解决方案**：放弃原生 confirm，对不涉及真实磁盘文件删除的操作（仅移出本地库记录），**直接执行即时删除（静默软删除）**，极大降低了交互摩擦力。

> **坑点 2：Vite 严格的 ES Modules 模块限制**
> **表现**：在重构图标映射代码时，顺手使用了 CommonJS 的 `require('lucide-react')`，导致 React 在运行时直接白屏崩溃。
> **原因**：Vite 默认只支持 ES6 `import`，遇到不可知的 `require` 会在浏览器抛出 ReferenceError。
> **解决方案**：坚决摒弃旧版 Webpack 的思维习惯，在 Vite 环境中**必须使用静态 `import`** 进行资源和组件引入。

> **坑点 3：React 复杂组件 (Radix UI) 的事件冒泡冲突**
> **表现**：DropdownMenu 中的 MenuItem 被点击后，如果不及时阻止默认行为，其内置的抢占焦点和自动关闭机制，会吞噬掉我们绑定的外部事件逻辑。
> **解决方案**：使用 `onSelect={(e) => e.preventDefault()}` 替代 `onClick`，从而接管事件控制权，保证我们的业务逻辑能稳定触达。

> **坑点 4：Dnd-kit 的“廉价网页感”陷阱**
> **表现**：刚实现拖拽时，被拖动的卡片只是变成了 50% 透明度并在原地挤压，视觉极其粗糙。
> **解决方案**：必须额外引入 `@dnd-kit/core` 的 `<DragOverlay>`。只有使用 Overlay 将被拖拽元素独立出 DOM 流，才能为其赋予高级的阴影、放大、毛玻璃等悬浮质感特效，同时保留原位置的虚线坑位占位符，这是实现桌面级手感的重中之重。
