# NoNo Harness Manager AI 架构与实施路线

## 1. 架构结论

NoNo Harness Manager 不是纯本地资产管理器，也不是内置一个独立 AI 系统。

**正确方向是分层：本地 Core 负责事实与安全，API Layer 暴露受控能力，Intelligence Layer 负责分析与提案，UI 负责确认与应用。**

所有外部 AI 客户端和未来内置 Copilot 都必须调用同一套 Harness API。

## 2. 四层架构

### 2.1 Harness Core

职责：

```text
本地资源扫描
SQLite 索引
权限和路径控制
使用统计
健康检查
proposal apply
audit log
```

约束：

```text
默认本地优先
默认只读
默认不联网
不自动上传敏感内容
所有写操作必须进入 audit_logs
```

### 2.2 Harness API Layer

职责：

```text
Tauri Commands
CLI
Harness MCP Server
```

原则：

```text
外部 AI 和内置 AI 调用同一套语义
AI 不能直接读写 SQLite
AI 不能直接读写本地文件
AI 只能通过受控工具读取安全上下文
MCP Server 第一阶段只开放只读工具和 proposal 工具
```

当前第一版 API 语义：

```text
list_harness_resources
get_harness_resource_context
create_intelligence_proposal
list_intelligence_proposals
apply_intelligence_proposal
```

### 2.3 Harness Intelligence Layer

职责：

```text
Skill 中文简介生成
MCP 功能解释
Memory 分类
Knowledge 摘要
Project 绑定建议
Health Check 修复建议
```

关键边界：

```text
Intelligence Layer 只能生成 proposal
proposal 默认 pending
用户确认后才允许 apply
敏感内容默认不送入 AI
除非用户明确确认，否则不联网
```

### 2.4 Harness UI

职责：

```text
展示资源
展示 AI proposal
让用户预览差异
让用户确认或拒绝
触发 apply
展示 audit log
```

UI 不直接绕过 API 写数据库或文件。

## 3. 数据模型方向

AI-ready 元数据字段：

```text
description
summary
category
tags
confidence
evidence_files
manual_override
last_analyzed_at
```

已落地资源：

```text
skills
mcp_servers
memory_sources
knowledge_bases
projects
```

治理表：

```text
intelligence_proposals
audit_logs
```

## 4. Proposal 工作流

```text
1. AI 通过 API 获取安全上下文
2. AI 生成 proposed_changes
3. API 写入 intelligence_proposals，状态为 pending
4. UI 展示 proposal
5. 用户确认
6. API apply proposal
7. Core 更新资源字段
8. Core 写 audit_logs
```

第一阶段 apply 只支持：

```text
skill
mcp_server
```

Memory、Knowledge、Projects 先具备表结构和 proposal 能力，后续再接真实页面和扫描逻辑。

## 5. MCP Server 第一阶段边界

开放工具：

```text
harness_list_resources
harness_get_resource_context
harness_create_proposal
```

明确不开放：

```text
任意命令执行
直接文件读取
直接数据库写入
apply proposal
删除资源
启动 Agent
真实 MCP 调用代理
```

## 6. 实施阶段

### Phase 1：AI-ready 数据底座

目标：

```text
资源具备 AI 可治理字段
proposal 表落地
audit log 表落地
Core 可以 apply proposal
```

验收：

```text
新装数据库 migration 能创建全部字段和表
Skills/MCP 能继续扫描和展示
AI 字段为空时不破坏现有 UI
apply proposal 会写 audit log
```

### Phase 2：Harness API / CLI / MCP

目标：

```text
外部 AI 客户端能读取 Harness 资源
外部 AI 客户端能创建 proposal
外部 AI 客户端不能直接 apply 或执行命令
```

验收：

```text
CLI 能 list/context/propose
MCP Server 能 tools/list 和 tools/call
MCP Server 只暴露只读和 proposal 工具
敏感 env 只以脱敏值出现
```

### Phase 3：内置轻量 Copilot

目标：

```text
Honeycomb 客户端内置 Copilot 面板
Copilot 调用同一套 Harness API
不另起数据库读写逻辑
不绕过 proposal/apply/audit
```

本阶段暂不实现。

### Phase 4：任务式 Agent

目标：

```text
批量整理 Skill 描述
批量解释 MCP
记忆体检
知识库摘要
项目绑定建议
```

本阶段暂不实现。
