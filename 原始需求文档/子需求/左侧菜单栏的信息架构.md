请优化 AppShell 左侧菜单栏的信息架构和视觉层级。

当前问题：
左侧菜单目前把 Dashboard、Agents、Skills、MCP、Proposals、Memory、Knowledge、Projects、Analytics、Health Check 全部平铺展示。虽然能用，但层级不够清晰，已完成模块和 Coming Soon 模块权重相同，Proposals 的位置也不太符合资源管理逻辑。

请按以下要求调整：

一、菜单分组

将左侧菜单改成分组结构：

Overview
- Dashboard
- Analytics
- Health Check

Assets
- Skills
- Agents
- MCP
- Memory
- Knowledge

Work
- Projects
- Proposals

底部：
- Settings

二、当前阶段权重

由于当前产品阶段重点是 Skills 资产管理，请在 Assets 分组中将 Skills 放在 Agents 前面：

Assets
- Skills
- Agents
- MCP
- Memory
- Knowledge

三、模块状态 badge

为导航项增加可选 status badge：

- Beta：已经可用但还在完善
- Soon：Coming Soon
- Ready：稳定可用，可后续再用
- Disabled：暂不可用

当前建议：
- Skills: Beta
- Agents: Beta
- MCP: Beta
- Proposals: Beta
- Memory: Soon
- Knowledge: Soon
- Projects: Soon
- Analytics: Soon
- Health Check: Soon

Dashboard 和 Settings 不需要 badge。

四、Coming Soon 模块视觉弱化

对于 status = Soon 的菜单项：
- 文字使用 muted color
- badge 显示 Soon
- 仍然允许点击进入 Coming Soon 页面
- 不要和 Beta 模块同等视觉权重

五、品牌区优化

顶部保持：
- NoNo Harness
- ThemeToggle

可以在标题下增加一行小字：
Local · v0.1

不要让顶部区域变高，保持简洁。

六、视觉要求

1. 分组标题使用小号 uppercase 或中文小标题，颜色 muted。
2. 每组之间留 12px 到 16px 间距。
3. 导航项高度保持 40px 到 44px。
4. 激活态清晰但不要过重。
5. badge 要小，不要抢主导航文本。
6. Settings 继续固定在底部。
7. 保持浅色 / 深色主题可读。
8. 不要改页面路由。
9. 不要改各业务页面。
10. 不要引入大型依赖。
