# **NoNo Harness Manager 项目初始化总提示词**

你现在要帮我初始化一个 macOS 桌面端项目，项目名称暂定为 `nono-harness-manager`。

这个项目不是单纯的 Skill 管理器，而是一个本地 Agent Harness 管理器。完整产品未来会包含 Agents、Skills、MCP、Memory、Knowledge、Projects、Analytics、Health Check 等模块。但第一阶段只开发 Skills 管理面板，同时搭好完整应用框架和后续模块占位。

## **一、技术栈约定**

请严格使用以下技术栈，不要擅自更换：

- 桌面框架：Tauri v2
- 前端框架：React + TypeScript
- 构建工具：Vite
- UI：Tailwind CSS + shadcn/ui
- 表格：TanStack Table
- 状态管理：Zustand
- 本地数据库：SQLite
- Rust 数据访问：SQLx
- 文件扫描：walkdir / ignore
- 元数据解析：serde_yaml / serde_json
- Markdown 预览：react-markdown + remark-gfm
- YAML 编辑器：CodeMirror 6
- 日志：tracing + tauri-plugin-log

如果某个依赖存在版本兼容问题，先说明原因和替代方案，不要悄悄替换技术栈。

## **二、产品边界约定**

完整框架包括：

1. Agents：管理 Agent 客户端，并支持一键启动，例如 Codex、Claude Code、Cursor、Qoder、ChatGPT Mac 客户端。
2. Skills：管理本机分散在不同目录中的 Prompt、脚本、工作流、能力包。
3. MCP：管理 MCP Server、Tools、Resources、Prompts、启动命令、环境变量和配置文件。
4. Memory：直接链接展示本地记忆目录，分类展示记忆，并提供记忆体检。
5. Knowledge：管理 Obsidian、VK、IMA 本地索引、项目知识库。
6. Projects：把 Agent、Skills、MCP、Memory、Knowledge 绑定到具体项目。
7. Analytics：统计资源使用情况。
8. Health Check：统一体检中心。

第一阶段只做：

- App Shell
- 左侧导航
- Dashboard 占位
- Agents 占位
- Skills 可用
- MCP 占位
- Memory 占位
- Knowledge 占位
- Projects 占位
- Analytics 占位
- Health Check 占位

第一阶段不做：

- AI 自动生成元数据

- Knowledge embedding
- 项目资源图谱

不要主动扩展这些功能。需要扩展时，必须先给出影响评估和是否进入后续阶段的建议。

## **三、安全原则约定**

这个项目管理本机文件、Agent、MCP、记忆和知识库，所以必须默认安全。

请遵守：

1. 默认本地优先，不联网。
2. 默认只读扫描，不执行脚本。
3. 不默认扫描全盘。
4. 不默认扫描用户 home 全目录。
5. 只扫描用户主动添加的目录源。
6. 前端不能获得无限制文件系统权限。
7. 所有本地文件操作必须通过 Rust Tauri command 封装。
8. 写入 `skill.yaml` 前必须校验 YAML。
9. 写入前必须备份原文件。
10. 敏感目录默认拒绝扫描，包括但不限于：
    - `~/.ssh`
    - `~/Library/Keychains`
    - `~/Library/Messages`
    - `~/Library/Mail`
    - `~/Library/Application Support/AddressBook`
11. 不允许把复杂 shell 命令作为默认启动方式。
12. Agent 一键启动第一阶段只允许使用 `open -a "AppName"` 或打开用户明确选择的 `.app` 路径。
13. 不记录文件正文到日志。
14. 不把用户本地路径、记忆内容、Skill 内容上传到任何远程服务。

## **七、Skills MVP 识别规则**

扫描 Skill 目录时，第一阶段按以下规则识别候选 Skill：

1. 文件夹内存在 `skill.yaml`，识别为标准 Skill。
2. 文件夹内存在 `skill.json`，识别为标准 Skill。
3. 文件夹内存在 `README.md`，识别为候选 Skill。
4. `.md` 文件识别为 Markdown / Prompt 型 Skill。
5. `.py`、`.sh`、`.js`、`.ts`、`.applescript` 识别为脚本型 Skill，但第一阶段绝不执行。
6. 文件名包含 `skill`、`prompt`、`workflow`、`agent`，识别为候选 Skill。
7. 扫描时忽略：
    - `node_modules`
    - `.git`
    - `dist`
    - `build`
    - `__pycache__`
    - `.venv`
    - `.DS_Store`
    - `target`
    - `coverage`
